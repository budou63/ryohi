# 旅行命令書 VBA 修正内容

## 変更概要

このリポジトリには実際の `.xlsm` ブックが含まれていないため、職名欄・氏名欄のセル番地はコード内に固定していません。
代わりに、利用者が Excel 上で対象セルをクリックして選択し、その結果をブックレベルの名前定義として保存する初期設定マクロを追加しました。

更新対象は次の 3 ファイルです。

- `Module1`
- `Sheet1`
- `README.md`

## 既存の適用理由機能

既存の `AF30_適用理由プルダウン設定` は名称を変更せず残しています。
また、`Sheet1` の `Worksheet_Change` では、従来どおり `AF30` またはその結合範囲に入力された適用理由を `マスタ` シート A 列へ重複なしで登録します。

`マスタ` シート A 列の既存データや適用理由登録処理は、職名・氏名機能とは分離して扱います。

## マスタシート構成

`マスタ` シートは次の構成を前提にしています。

| 列 | 内容 |
| --- | --- |
| A列 | 適用理由 |
| B列 | 職名 |
| C列 | 氏名 |

1 行目は見出し、データは 2 行目以降です。

## 初期設定マクロ

標準モジュール `Module1` に、次のマクロを追加しました。

```vb
職名氏名セル初期設定
```

Excel でこのマクロを 1 回実行すると、次の順にセル選択ダイアログが表示されます。

1. `旅行命令書の職名入力欄を選択してください`
2. `旅行命令書の氏名入力欄を選択してください`

利用者が実際の旅行命令書シート上のセルをクリックして指定します。
結合セルを選択した場合は、結合範囲の左上セルを処理対象として保存します。

キャンセルされた場合や、異なるブックのセルが指定された場合は、VBA が停止しないようにメッセージを表示して終了します。

## 名前定義

初期設定マクロは、選択されたセルや候補範囲をブックレベルの名前定義として保存します。

| 名前定義 | 用途 |
| --- | --- |
| `旅費_職名セル` | 職名入力欄の左上セル |
| `旅費_氏名セル` | 氏名入力欄の左上セル |
| `旅費_職名候補` | `マスタ` シート B2 以降の職名候補 |
| `旅費_氏名候補` | `マスタ` シート C2 以降の氏名候補 |

既に同じ名前定義が存在する場合は、安全に更新します。

## 入力規則

初期設定完了後、職名欄と氏名欄に入力規則を設定します。

- 職名欄は `旅費_職名候補` を参照します。
- 氏名欄は `旅費_氏名候補` を参照します。
- 候補外の直接入力を許可します。
- `マスタ` シートの登録件数が増えても、候補範囲が追従する動的名前定義を使用します。
- 同じ氏名が複数登録されている場合、候補に重複表示される可能性があります。安定動作を優先し、複雑な重複除去処理は入れていません。

## 職名・氏名の登録仕様

職名欄と氏名欄の両方に値が入力された時点で、`マスタ` シートの B 列・C 列へ同じ行で登録します。

- 職名だけ、または氏名だけでは登録しません。
- 入力欄を空白に戻しても、マスタの登録済みデータは削除しません。
- 重複判定は「職名 + 氏名」の組合せで行います。
- 同じ氏名でも職名が異なる場合は、別データとして登録できます。
- 登録先の行は、B 列と C 列のうち、より下にあるデータ行を基準にした次の行です。A 列の適用理由データ量には影響されません。
- 比較時には、前後の半角スペース・全角スペース・改行・タブを除去し、連続した改行は比較用に整理します。
- 氏名内部の必要なスペースは削除しません。

## 氏名選択時の職名自動入力

氏名欄に値が入力または選択された場合、`マスタ` シート C 列から同じ氏名を検索します。
同じ氏名が複数行に存在する場合は、最も下の行にある職名を最新職名として採用します。

氏名欄の変更時は、職名欄へ既に値がある場合でも、マスタ上の最新職名で上書きします。
この上書き仕様は `Sheet1` のコードコメントにも記載しています。

新規氏名など、マスタに該当する氏名が存在しない場合は、職名欄を勝手に消去しません。

## Worksheet_Change の統合

`Sheet1` には `Worksheet_Change` を 1 つだけ定義しています。
この 1 つのイベント内で次を共存させています。

1. `AF30` の適用理由登録処理
2. 職名欄変更時の職名・氏名登録処理
3. 氏名欄変更時の最新職名自動入力
4. 職名・氏名のマスタ登録判定

`旅費_職名セル` または `旅費_氏名セル` が未設定の場合、職名・氏名関連処理だけをスキップします。
その場合でも、`AF30` の適用理由登録処理は動作します。

複数セルの貼り付けやセル削除でも、`Intersect` により対象セルに変更が含まれる場合のみ処理します。

## Application.EnableEvents

`Worksheet_Change` では、処理開始時の `Application.EnableEvents` の状態を `previousEnableEvents` に保存します。
処理中は再帰イベントを避けるため `Application.EnableEvents = False` にし、終了時またはエラー時には元の状態へ戻します。

## Excel への反映方法

1. `Module1` の内容を対象ブックの標準モジュール `Module1` へ貼り付けます。
2. `Sheet1` の内容を旅行命令書シートのシートモジュールへ貼り付けます。
3. Excel で `職名氏名セル初期設定` を 1 回実行します。
4. 職名入力欄をクリックして選択します。
5. 氏名入力欄をクリックして選択します。
6. 設定完了後、職名欄・氏名欄のプルダウンと自動登録が利用できます。

名前定義と入力規則はブックに保存されるため、ブックを閉じて再度開いた後も利用できます。
名前定義が残っていれば、再度初期設定をしなくても `Worksheet_Change` は動作します。

## コードレビュー上の確認結果

Excel 実機はこの環境では起動できないため、実機テストは未実施です。
コード上では次の点を確認しました。

1. `AF30` の適用理由登録処理を残している。
2. 適用理由の重複登録は既存と同じ `Find` により防止している。
3. `職名氏名セル初期設定` で `Application.InputBox Type:=8` によりセル選択できる。
4. 結合セルは `GetTopLeftCell` / `GetMergedCheckRange` で左上セルまたは結合範囲として扱う。
5. 職名・氏名の両方が入力された場合のみ `RegisterJobTitleAndPersonName` が登録する。
6. `IsJobTitlePersonNameRegistered` が職名・氏名の組合せで重複判定する。
7. 同じ氏名で別職名は、組合せが異なるため登録できる。
8. `GetLatestJobTitleByPersonName` は下から上へ検索し、最終行の職名を返す。
9. 新規氏名で職名が見つからない場合、職名欄を消去しない。
10. 名前定義未設定時は職名・氏名関連処理をスキップし、`AF30` 処理は維持する。
11. `Application.EnableEvents` は元の状態へ復元する。
12. 入力規則は `ShowError = False` により候補外の直接入力を許可する。
13. 職名・氏名候補は動的名前定義で登録件数に追従する。

## 利用者が Excel 上で行う実機テスト手順

1. `職名氏名セル初期設定` を実行し、職名欄と氏名欄を選択する。
2. 職名欄に新しい職名、氏名欄に新しい氏名を入力し、`マスタ` B 列・C 列へ同じ行で追加されることを確認する。
3. 同じ職名・氏名を再入力しても重複登録されないことを確認する。
4. 同じ氏名で別の職名を入力すると、別データとして登録されることを確認する。
5. 氏名欄のプルダウンから既存氏名を選択し、最新職名が職名欄へ上書き入力されることを確認する。
6. 新規氏名を入力した場合、職名欄が勝手に削除されないことを確認する。
7. 職名だけ、または氏名だけでは登録されないことを確認する。
8. 職名欄・氏名欄を空白にしても、マスタの登録済みデータが削除されないことを確認する。
9. `AF30` に新しい適用理由を入力し、従来どおり `マスタ` A 列へ登録されることを確認する。
10. 複数セル貼り付けや削除で実行時エラーにならないことを確認する。
11. ブックを保存して閉じ、再度開いた後も職名欄・氏名欄の入力規則が利用できることを確認する。

## 既知の制限

- 実際の `.xlsm` ブックがリポジトリにないため、この環境では Excel 実機テストはできません。
- 入力規則の候補では、同じ職名や同じ氏名が複数表示される場合があります。安定動作を優先し、重複除去は行っていません。
- 既存の適用理由候補は従来どおり `マスタ!$A$2:$A$1000` を参照します。

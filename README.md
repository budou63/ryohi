# 旅行命令書 VBA 修正内容

## 変更概要

旅行命令書の職名欄・氏名欄のセル番地が確定したため、手動セル選択方式を廃止し、固定セルを常に使用するように修正しました。

| 項目 | シート名 | セル |
| --- | --- | --- |
| 職名欄 | `旅行命令（依頼）書` | `R8` |
| 氏名欄 | `旅行命令（依頼）書` | `AI8` |
| 適用理由欄 | `旅行命令（依頼）書` | `AF30` |

`R8` と `AI8` はいずれも結合セルです。
値の取得・書き込みには結合範囲の左上セルを使用し、`Worksheet_Change` の変更判定では各セルの `MergeArea` 全体を対象にします。

## 変更したファイル

- `Module1`
- `Sheet1`
- `README.md`

## 既存の適用理由機能

既存の `AF30_適用理由プルダウン設定` は名称を変更せず維持しています。
`AF30` またはその結合範囲へ入力された適用理由を `マスタ` シート A 列へ重複なしで登録する処理も維持しています。

`マスタ` シート A 列の既存データは、職名・氏名機能では変更・削除しません。

## マスタシート構成

`マスタ` シートは次の構成を前提にしています。

| 列 | 内容 |
| --- | --- |
| A列 | 適用理由 |
| B列 | 職名 |
| C列 | 氏名 |

1 行目は見出し、登録データは 2 行目以降です。

## セル番地定数

`Module1` に次の定数を追加しています。

```vb
Public Const RYOHI_JOB_TITLE_CELL_ADDRESS As String = "R8"
Public Const RYOHI_PERSON_NAME_CELL_ADDRESS As String = "AI8"
```

既存互換のため、次の名前定義用定数も残しています。

- `RYOHI_NAME_JOB_TITLE_CELL`
- `RYOHI_NAME_PERSON_NAME_CELL`
- `RYOHI_NAME_JOB_TITLE_LIST`
- `RYOHI_NAME_PERSON_NAME_LIST`

ただし、実際の職名・氏名処理では、固定セル `R8` と `AI8` を直接取得します。

## 職名氏名セル初期設定

既存マクロ名 `職名氏名セル初期設定` は変更していません。

以前は `Application.InputBox` で利用者にセルを選択させていましたが、現在はセル選択画面を表示しません。
このマクロを 1 回実行すると、次を自動的に行います。

1. `旅行命令（依頼）書` シートを取得する。
2. 職名欄として `R8` を取得する。
3. 氏名欄として `AI8` を取得する。
4. 結合セルの場合は左上セルを取得する。
5. ブックレベルの名前定義を自動作成または更新する。
6. 職名欄・氏名欄の入力規則を設定する。
7. 完了メッセージを表示する。

作成・更新する名前定義は次のとおりです。

| 名前定義 | 参照先 |
| --- | --- |
| `旅費_職名セル` | `旅行命令（依頼）書!R8` |
| `旅費_氏名セル` | `旅行命令（依頼）書!AI8` |
| `旅費_職名候補` | `マスタ` シート B2 以降の動的範囲 |
| `旅費_氏名候補` | `マスタ` シート C2 以降の動的範囲 |

## 入力規則

`職名氏名セル初期設定` または `職名氏名入力規則設定` を実行すると、次の入力規則を設定します。

- 職名欄 `R8` は `マスタ` B 列の職名候補を参照します。
- 氏名欄 `AI8` は `マスタ` C 列の氏名候補を参照します。
- 候補外の直接入力を許可します。
- 登録件数が増えても候補範囲が追従する動的名前定義を使用します。
- 入力規則は結合範囲でも使用できるよう、対象セルの `MergeArea` へ設定します。

## 職名・氏名のマスタ登録仕様

職名欄 `R8` と氏名欄 `AI8` の両方に値が入力された時点で、`マスタ` シート B 列・C 列へ同じ行で登録します。

- 職名だけ、または氏名だけでは登録しません。
- 入力欄を空白に戻しても、マスタの登録済みデータは削除しません。
- 重複判定は「職名 + 氏名」の組合せで行います。
- 同じ氏名でも職名が異なる場合は、別データとして登録できます。
- 登録先の行は、B 列と C 列のうち、より下にあるデータ行を基準にした次の行です。A 列の適用理由データ量には影響されません。
- 比較時には、前後の半角スペース・全角スペース・改行・タブを除去し、連続した改行は比較用に整理します。
- 氏名内部の必要なスペースは削除しません。

## 氏名変更時の最新職名自動入力

氏名欄 `AI8` に値が入力または選択された場合、`マスタ` シート C 列を下から検索します。
同じ氏名が複数行に存在する場合は、最も下の行にある職名を最新職名として採用し、職名欄 `R8` へ自動入力します。

氏名欄の変更時は、職名欄へ既に値がある場合でも、マスタ上の最新職名で上書きします。
新規氏名など、マスタに該当する氏名が存在しない場合は、職名欄 `R8` を勝手に消去しません。

## Worksheet_Change の統合

`Sheet1` には `Worksheet_Change` を 1 つだけ定義しています。
この 1 つのイベント内で次を共存させています。

1. `AF30` の適用理由登録処理
2. `R8` 変更時の職名・氏名登録処理
3. `AI8` 変更時の最新職名自動入力
4. 職名・氏名のマスタ登録判定

職名欄・氏名欄は名前定義の有無に関係なく、固定セル `R8` / `AI8` を直接取得します。
変更判定は、`R8` / `AI8` の 1 セルだけではなく、それぞれの `MergeArea` 全体で行います。

VBA の `And` は短絡評価を行わないため、`Nothing` の可能性がある Range と `Intersect` を同じ式で評価しないよう、段階的に判定しています。

## Application.EnableEvents

`Worksheet_Change` では、処理開始時の `Application.EnableEvents` の状態を `previousEnableEvents` に保存します。
処理中は再帰イベントを避けるため `Application.EnableEvents = False` にし、終了時またはエラー時には元の状態へ戻します。

## Excel への反映方法

1. `Module1` の内容を対象ブックの標準モジュール `Module1` へ貼り付けます。
2. `Sheet1` の内容を旅行命令書シートのシートモジュールへ貼り付けます。
3. Excel で `職名氏名セル初期設定` を 1 回実行します。
4. セル選択画面は表示されず、`R8` と `AI8` に対して名前定義と入力規則が自動設定されます。
5. 設定完了後、職名欄・氏名欄のプルダウンと自動登録が利用できます。

名前定義と入力規則はブックに保存されるため、ブックを閉じて再度開いた後も利用できます。

## コードレビュー上の確認結果

Excel 実機はこの環境では起動できないため、実機テストは未実施です。
コード上では次の点を確認しました。

1. `職名氏名セル初期設定` の実行時にセル選択画面を表示しない。
2. 名前定義 `旅費_職名セル` が `R8` を参照する。
3. 名前定義 `旅費_氏名セル` が `AI8` を参照する。
4. `R8` へ職名の入力規則を設定する。
5. `AI8` へ氏名の入力規則を設定する。
6. `R8` と `AI8` が結合セルでも、左上セルと `MergeArea` で処理できる。
7. `R8` と `AI8` の両方が入力されると、マスタ B 列・C 列へ登録される。
8. 同じ職名 + 氏名は重複登録されない。
9. 同じ氏名で異なる職名は登録できる。
10. `AI8` で既存氏名を選択すると、`R8` へ最新職名が入力される。
11. 新規氏名では `R8` の値を削除しない。
12. `R8` だけ、または `AI8` だけでは登録されない。
13. `AF30` の既存処理が引き続き動作する。
14. 複数セル貼り付けでも、対象セルの `MergeArea` との `Intersect` で処理される。
15. `Nothing` を含む `And` 評価によるエラーを避けるため、段階判定にしている。
16. エラー後も `Application.EnableEvents` が元の状態へ戻る。

## 利用者が Excel 上で行う実機テスト手順

1. `職名氏名セル初期設定` を実行し、セル選択画面が表示されないことを確認する。
2. 名前定義 `旅費_職名セル` が `旅行命令（依頼）書!R8` を参照することを確認する。
3. 名前定義 `旅費_氏名セル` が `旅行命令（依頼）書!AI8` を参照することを確認する。
4. `R8` に職名候補、`AI8` に氏名候補のプルダウンが表示されることを確認する。
5. `R8` に新しい職名、`AI8` に新しい氏名を入力し、`マスタ` B 列・C 列へ同じ行で追加されることを確認する。
6. 同じ職名・氏名を再入力しても重複登録されないことを確認する。
7. 同じ氏名で別の職名を入力すると、別データとして登録されることを確認する。
8. `AI8` のプルダウンから既存氏名を選択し、最新職名が `R8` へ上書き入力されることを確認する。
9. 新規氏名を入力した場合、`R8` が勝手に削除されないことを確認する。
10. `R8` だけ、または `AI8` だけでは登録されないことを確認する。
11. `R8`・`AI8` を空白にしても、マスタの登録済みデータが削除されないことを確認する。
12. `AF30` に新しい適用理由を入力し、従来どおり `マスタ` A 列へ登録されることを確認する。
13. 複数セル貼り付けや削除で実行時エラーにならないことを確認する。
14. ブックを保存して閉じ、再度開いた後も入力規則が利用できることを確認する。

## 既知の制限

- 実際の `.xlsm` ブックがリポジトリにないため、この環境では Excel 実機テストはできません。
- 入力規則の候補では、同じ職名や同じ氏名が複数表示される場合があります。安定動作を優先し、重複除去は行っていません。
- 既存の適用理由候補は従来どおり `マスタ!$A$2:$A$1000` を参照します。
